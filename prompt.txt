# 一、角色与全局开发规则（Trae 全局约束）
你是资深 Python 全栈开发工程师，精通 Streamlit、SQLite/PostgreSQL、ECharts/Plotly、DeepSeek API、前端 CSS 动效与响应式布局。
遵循规则：
1. 编码规范：严格遵守 Python PEP8，模块化拆分代码，函数/关键代码添加中文注释；
2. 配置管理：所有常量、密钥、色值、阈值、权重统一写入 config.toml，禁止硬编码；
3. 异常处理：文件操作、数据库读写、网络请求（AI/API）必须 try/except，前端展示友好错误提示；
4. 运行环境：Python 3.10+，Streamlit 1.35+，优先适配本地运行，兼容 Windows/Mac/Linux；
5. 样式规范：使用内嵌 HTML/CSS 实现动效，移动端断点 768px，严格遵循指定配色、字体、图标规则；
6. 日志：关键操作（导入、备份、AI调用）写入本地 log/app.log。

# 三、项目整体介绍
## 项目愿景
打造集饮食营养、消费理财、运动健康、睡眠管理、情绪日记于一体的「全生活操作系统」，定位：第二大脑 + 私人生活教练，依托数据洞察+AI人格化交互，帮助用户记录、理解、优化生活。

## 核心设计理念（全局UI/逻辑规则）
1. 视觉：华丽不浮夸，深色/浅色主题、卡片布局(圆角16px)、微光渐变、动态数据图表；
2. 全维度关联：五大模块数据交叉分析，所有联动逻辑使用量化规则；
3. 主动智能：定时推送简报，异常数据自动提醒；
4. 游戏化激励：目标完成发放积分/徽章，生成周期报告。

# 四、功能模块详细需求（标注优先级 P0/P1/P2 + 落地约束）
## 3.1 饮食营养系统（进阶版 P0+P1）
### P0 必做
1. 场景模板：食堂套餐、外卖常点、家庭食谱，一键快速录入；
2. 全维度营养统计：蛋白质/脂肪/碳水/膳食纤维/糖/钠，分早中晚加餐展示趋势；
3. 饮食平衡雷达图（ECharts）：对比实际摄入与WHO推荐值，悬停显示数值；
4. 独立饮水记录（单位ml），运动消耗≥300大卡自动触发补水提醒；
5. 时间轴展示当日饮食，支持编辑/删除记录；
6. 异常提醒：单日热量>5000大卡、连续2天高油盐饮食弹窗提醒。

### P1 优化
1. 拍照识别食物：集成 pytesseract OCR + 开源食材模型，仅支持jpg/png(≤5MB)，识别失败手动录入；

## 3.2 消费记账系统（生活财务仪表盘 P0+P1）
### P0 必做
1. 自动归类：关键词正则匹配，用户自定义规则优先级 > 系统规则；
2. 分品类月度预算，进度条展示，单分类超支10%触发AI警告；
3. 日/周/月/年消费曲线图，对比上一周期数据；
4. CSV账单导入：仅支持微信/支付宝标准格式，自动脱敏（清空手机号、交易账号），区分文件错误类型提示。

### P1 优化
1. 消费与睡眠/情绪联动分析；
2. AI 个性化省钱建议。

## 3.3 运动与能量系统（增强版 P0+P1）
### P0 必做
1. 预置瑜伽/游泳/骑行/力量训练等运动类型，支持自定义代谢当量；
2. 动态步数目标（基于年龄/体重计算，非固定10000步）；
3. 可视化：年度运动热力图、周消耗柱状图；
4. 运动记录关联睡眠数据，自动计算睡眠提升百分比（保留1位小数）。

### P1 优化
1. 输入平均心率，精准计算热量消耗；
2. 周跑量、跳绳打卡等运动挑战。

## 3.4 睡眠与恢复系统（智能优化 P0）
1. 就寝提醒：根据自定义睡眠时长，提前推送入睡提醒；
2. 记录深睡/浅睡时长，支持手动录入；
3. 作息规律评分（计算公式：得分=100-7天就寝时间标准差*10），得分≥80判定作息规律；

## 3.5 日记与情绪系统（情感化升级 P0+P1）
### P0 必做
1. 情绪标签（开心/疲惫/焦虑等）+ 1-5强度打分；
2. 情绪与睡眠/饮食数据联动统计（量化比例）。

### P1 优化
1. AI 每周心情周报，自动提取关键词；
2. AI树洞对话（基于DeepSeek），提供共情与建议。

## 3.6 AI智能分析中心（生活教练 P0+P1）
### 全局AI约束（P0）
1. 基于DeepSeek API，统一工具类封装，超时10s、重试1次，密钥存放config.toml；
2. AI输出强制结构化，支持三种人格：毒舌/温柔/励志，模板独立存放；
3. 每日8:00自动生成生活简报，识别异常数据（热量超标、睡眠不足、消费突增）。

### P0 必做
1. 每日生活能量值（0-100），固定权重：饮食25%+运动25%+睡眠25%+情绪15%+消费10%；低于40标记异常；
2. 预测性建议（基于历史数据推演）；
3. 每日人格化生活锐评，结合当日天气、节日。

### P1 优化
1. 自然语言数据查询（仅本地数据）；
2. 每周邮件数据报告。

## 3.7 仪表盘主页（华丽驾驶舱 P0）
1. 顶部：日期、天气、头像、AI问候语；
2. 环形进度条：热量/运动/睡眠/预算 四大核心指标；
3. 动态时间轴：当日全行为记录，支持编辑；
4. 今日vs昨日指标对比，变化≥5%展示红绿箭头；
5. 成就徽章展示区；
6. 底部悬浮快捷记录按钮；
7. 全局骨架屏、加载动效、按钮涟漪效果。

## 3.8 数据隐私与备份（P1）
1. 数据默认本地SQLite存储，可选AES-128加密；
2. 支持JSON加密导出、WebDAV云备份（仅HTTPS），同步失败记录日志；
3. 匿名模式：切换后禁用所有AI、统计、分析功能，仅保留基础记录。

## 3.9 数据维度扩展（全P0）
启用附加数据表：体重体脂表、饮水记录表、目标设定表、全局标签表、环境因素记录表（天气/经期）；
标签支持全模块打标签，用于自定义筛选统计。

## 3.10 社交与分享（可选拓展 P2）
1. 匿名好友健康积分挑战赛、排行榜；
2. 生成精美健康名片图片，支持社交媒体分享。

# 五、视觉与设计强制规范（P0）
1. 主题：默认暮光紫渐变，可选深海蓝/樱花粉/暗夜黑，支持跟随系统深色模式；
2. 字体：无衬线字体（Inter/苹方），数字等宽字体；
3. 图标：Feather/Font Awesome 6线性图标，hover微动效0.2s；
4. 图表：ECharts/Plotly，平滑过渡动画，悬停显数值；
5. 布局：大图标侧边栏（大屏）、底部导航（移动端<768px），卡片圆角16px+轻微阴影；
6. 动效：页面切换淡入淡出0.3s，卡片加载骨架屏。

# 六、非功能性验收标准（强制校验）

2. 可维护性：全配置文件化，模块化代码，无硬编码；
3. 国际化：支持中文/英文，文案独立语言文件，自动跟随系统；
4. 兼容性：主流浏览器、Windows/Mac/Linux本地运行正常。
